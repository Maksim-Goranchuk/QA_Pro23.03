Инструмент для QA, который автоматически генерирует тестовые данные для полей ввода. Вместо того чтобы вручную придумывать «неудобные» значения, вы настраиваете тип поля и получаете готовый набор кейсов: от граничных значений до SQL-инъекций.

Файл test_data_generator.html — это самодостаточное веб-приложение. Никакой установки не нужно.

- Скачайте файл
- Откройте браузер (Chrome, Firefox, Edge) 
- Нажмите Ctrl+O → найдите файл → нажмите «Открыть»

Либо просто перетащите файл мышкой прямо в окно браузера. 

Тип поля — выберите, что именно тестируете: текст, email, пароль, число, дата, телефон, URL или логин. Под каждый тип подобраны специфичные кейсы.
Минимальная / максимальная длина — укажите ограничения вашего поля. Граничные значения будут сгенерированы точно под них: «ровно минимум», «минимум − 1», «ровно максимум», «максимум + 1» и так далее.
Категории — можно выбрать одну или несколько (зажмите Ctrl для мультивыбора):

Позитивные значения — корректные данные, которые поле должно принять
Граничные значения — крайние точки допустимого диапазона
Негативные / некорректные — пустые строки, пробелы, null, неверные типы
Спецсимволы и кодировки — кириллица, emoji, unicode, управляющие символы
Безопасность — XSS, SQL-инъекции, path traversal, template injection

Язык / локаль — влияет на то, какие символы используются при случайной генерации: смешанный, только латиница, только кириллица, или unicode-экзотика.
Степень рандомизации — определяет, насколько уникальными будут данные при каждом нажатии «Сгенерировать»:

Низкая — только перемешивание порядка значений
Средняя — часть значений генерируется случайно каждый раз
Высокая — максимум уникальности, генерируются дополнительные случайные варианты


Как пользоваться:
Выберите тип поля и укажите его ограничения по длине
Выберите нужные категории
Нажмите ⚡ Сгенерировать
Каждое значение можно скопировать кнопкой «копировать»
Нажмите ↓ Экспорт CSV, чтобы сохранить всё в файл для Excel или тест-кейсов

Значения с фиолетовым бейджем rand — сгенерированы случайно и будут другими при следующем запуске.
В нижней части кнопки генерации отображается seed — числовой идентификатор текущего набора. Если нужно воспроизвести конкретный набор данных — сохраните seed (он также попадает в CSV при экспорте).
